# 初版 Web Demo 实现说明 v1.0

## 实现状态

ChargeBI 初版 Web Demo 已完成基础可运行版本。

当前能力：

- 可运行 Next.js Web 应用
- 可生成 SQLite 示例数据库
- 可展示上海都市圈充电网络示例地图
- 可展示运营指标卡
- 可输入自然语言问题
- 可点击推荐问题
- 可返回问题理解、AI 工作流、业务结论、KPI、图表、表格、Schema 匹配和 SQL
- 可进行地图区域或站点选择，并作为上下文传入问数接口
- 可拦截隐私敏感问题
- 可引导超范围问题

## 运行方式

安装依赖：

```bash
npm install
```

生成数据库：

```bash
npm run db:generate
```

启动开发服务：

```bash
npm run dev
```

访问：

```text
http://localhost:3000
```

构建验证：

```bash
npm run build
```

## 已实现 API

### GET /api/overview

文件：

- `app/api/overview/route.ts`

用途：

- 返回最近 30 天全局 KPI
- 返回区域聚合数据

### GET /api/stations

文件：

- `app/api/stations/route.ts`

用途：

- 返回站点点位
- 返回站点最近 30 天收入、订单、利用率、故障和健康度

### POST /api/query

文件：

- `app/api/query/route.ts`

用途：

- 接收自然语言问题
- 执行规则兜底问数链路
- 返回结构化回答

## 当前技术取舍

### 规则兜底优先

初版未直接接入 DeepSeek/Kimi，而是实现规则兜底问数链路。

原因：

- 当前没有 API Key 和模型配置
- 作品集初版需要稳定可体验
- 规则链路能完整展示产品结构
- 后续可替换 `lib/query/orchestrator.ts` 中的意图识别和 SQL 生成层

### 地图为示意地图

初版地图不是 Leaflet 或高德底图，而是基于示例坐标的产品化示意地图。

原因：

- 避免地图 Key、瓦片服务和网络不稳定
- 更贴合个人网站视觉
- 足以展示地图点位、区域选择和结果高亮

后续如需增强真实地图，可替换 `ChargeMap` 组件。
